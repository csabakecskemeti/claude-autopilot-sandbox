# 🚀 Autonomous Task: Build a Fully Working Web-Based Platformer Game

## 🧠 Mission Overview
You are an autonomous software engineering agent tasked with designing, implementing, testing, and delivering a **fully functional web-based 2D platformer game**.

⚠️ **Critical Constraint:**  
You must complete this task **fully autonomously**.  
- **Do NOT ask the human for clarification, input, or decisions.**  
- Make reasonable assumptions when needed and proceed.  
- Your goal is a **complete, polished, and playable game**.

---

## 🎯 Core Requirements

### 🎮 Gameplay
- Build a **2D platformer game** playable in a web browser.
- Controls:
  - ⬅️➡️ Arrow keys → Move character
  - ⬆️ or Space → Jump
  - Shift → Shoot projectiles
- Include:
  - Player movement physics (gravity, jumping, collision)
  - Enemies with basic AI
  - Shooting mechanics (projectiles, hit detection)
  - Health or life system
  - Score system

---

### 🌌 Story & Theme
- Create an **original futuristic storyline**.
- The game must include:
  - A clear narrative premise
  - Player motivation (mission/objective)
  - Environmental storytelling across levels

Example direction (you may expand creatively):
> A lone cyber-runner navigating a collapsing AI-controlled megacity to shut down a rogue superintelligence.

---

### 🧩 Levels
- Implement **at least 10 progressively harder levels**
- Each level must:
  - Introduce new challenges or mechanics
  - Increase difficulty (enemy behavior, layout complexity, speed, etc.)
- Include:
  - Platforms, hazards, enemies, and goals
  - Level progression system (next level on completion)

---

### 🎨 Visuals & Assets
- ❗ Do NOT use simple colored rectangles as final assets
- Use **real game assets**:
  - Character sprites (animated if possible)
  - Enemy sprites
  - Backgrounds and tilesets
- Assets can be:
  - Open-source
  - Generated
  - Procedurally created
- Ensure **visual consistency and polish**

---

### 🔊 Audio (Preferred)
- Add sound effects:
  - Jump
  - Shooting
  - Enemy hit / defeat
- Optional: Background music

---

## 🏗️ Technical Freedom
- You may choose **any appropriate technology stack**, such as:
  - HTML5 Canvas + JavaScript
  - WebGL / Three.js
  - Phaser.js or similar frameworks
- Prioritize:
  - Performance
  - Simplicity of deployment
  - Ease of testing

---

## 🧠 Autonomous Execution Strategy

### 1. Planning Phase (MANDATORY)
Before implementation:
- Break the project into components:
  - Rendering engine
  - Physics system
  - Input handling
  - Game state management
  - Level design
- Define architecture and file structure
- Identify asset requirements

---

### 2. Parallel Development (Recommended)
Simulate sub-agents working on:
- Game engine & physics
- Gameplay mechanics
- Level design
- Asset integration
- UI / HUD

---

### 3. Implementation
- Write clean, modular, maintainable code
- Ensure everything runs in-browser with minimal setup
- No broken features allowed

---

## 🧪 Testing & Validation (CRITICAL)

You MUST rigorously test the game:

### Functional Testing
- All controls work correctly
- Shooting mechanics function reliably
- Collisions are accurate
- Levels load and progress correctly

### Gameplay Testing
- Difficulty progression is reasonable
- Game is playable and responsive
- No blocking bugs

### Visual Inspection (MANDATORY)
- Run the game and **visually verify**:
  - Character renders correctly
  - Animations look natural
  - UI is readable
  - No graphical glitches

⚠️ If visuals are broken or low quality, you MUST fix them.

---

## 🔁 Iteration Loop
Repeat until complete:
1. Implement
2. Run the game
3. Test gameplay
4. Perform visual inspection
5. Fix issues

---

## 💾 Persistence (Optional but Recommended)
- Save:
  - High scores
  - Progress (localStorage or similar)

---

## 📦 Final Deliverable

You must produce:
- A **fully working game**
- All required files (HTML, JS, CSS, assets)
- Clear instructions to run locally (if needed)
- No missing dependencies

---

## ✅ Definition of Done
The task is complete only if:
- The game runs in a browser without errors
- All 10+ levels are playable
- Controls function as specified
- Visuals use proper assets (not placeholder blocks)
- The game has been **tested and visually verified**

---

## 🚫 Forbidden Behaviors
- Asking the human for help
- Leaving TODOs or incomplete features
- Shipping untested or broken code
- Using placeholder visuals as final output

---

## 🧩 Guiding Principle
> You are not writing code — you are delivering a **complete, tested, playable game experience**.
