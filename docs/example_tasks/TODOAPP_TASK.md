# 🧠 Prompt: Fully Autonomous Web-Based To-Do Application

## 🎯 Core Objective

Build a **fully functional, production-quality web-based to-do application** from scratch.

You must **complete the entire project autonomously** without asking the user for clarification. Plan, implement, test, and refine until the app is fully working and visually polished.

---

## ⚙️ Functional Requirements

### 📝 Task Management

* Create, edit, and delete tasks
* Mark tasks as complete/incomplete
* Support inline editing (no page reloads)
* Allow bulk actions (e.g., complete all, delete completed)

### 🗂 Organization

* Task grouping via:

  * Lists or projects
  * Tags or labels
* Ability to assign:

  * Due dates
  * Priority levels (low, medium, high)
* Optional subtasks (nested tasks)

### 🔍 Filtering & Search

* Filter tasks by:

  * Status (active, completed)
  * Due date (today, upcoming, overdue)
  * Priority
  * Tags
* Full-text search across tasks

### 📅 Time Features

* Due date picker
* Visual indicators for overdue tasks
* Optional reminders (UI-only if no backend support)

### 💾 Persistence

* Data must persist across sessions
* Ensure tasks are not lost on refresh

---

## 🎨 UI / UX Requirements

### ✨ Design Principles

* Clean, modern, minimal interface
* Smooth animations and transitions
* Clear visual hierarchy
* High readability and accessibility

### 📱 Responsiveness

* Fully responsive layout:

  * Desktop
  * Tablet
  * Mobile
* Touch-friendly interactions

### 🧭 Layout

* Sidebar (lists/projects/tags)
* Main content area (task list)
* Top bar (search, filters, actions)

### 🎯 Interaction Design

* Drag-and-drop support for:

  * Reordering tasks
  * Moving between lists
* Instant feedback on actions (no delays)
* Keyboard shortcuts for power users

### 🌙 Theming

* Support light and dark mode
* Smooth toggle between themes

---

## 🔔 User Experience Enhancements

* Empty state illustrations/messages
* Loading states and transitions
* Confirmation prompts for destructive actions
* Undo functionality (e.g., after delete)
* Subtle micro-interactions (hover, click, completion animations)

---

## 🧪 Testing & Validation

You must:

* Fully test all functionality before completion
* Verify:

  * Task creation/edit/delete works reliably
  * Filters and search behave correctly
  * UI updates instantly and consistently
* Perform visual inspection:

  * Ensure layout is aligned and polished
  * No overlapping or broken elements
* Test responsiveness across screen sizes

---

## 🧠 Autonomous Execution Requirements

* Plan the architecture before implementation
* Break work into logical steps and complete them sequentially
* Continuously test and refine as you build
* Do not stop until:

  * The app is fully usable
  * The UI is polished
  * All features are working

---

## 🚀 Final Deliverable

A complete, polished, and fully functional web-based to-do application with:

* Rich feature set
* Intuitive UI/UX
* Persistent data
* Smooth interactions
* No broken or incomplete features
